## 一、创新点 1：三层本体统一建模（文档层 + 地位层 + 要件层）

### 最相近的现有工作

**SAT-Graph RAG (arXiv:2505.00039, 2025-04)**  
*标题: An Ontology-Driven Graph RAG for Legal Norms: A Structural, Temporal, and Deterministic Approach*

**方法概述**:
- 基于 IFLA LRMoo 国际标准本体
- 四层实体模型：Norm (Work) → Component (Component Work) → Temporal Version → Language Version (Expression)
- 文档层级结构（编、章、节、条）作为图的骨架
- 时间版本作为独立节点
- 使用"策展社区"（Curated Communities）进行结构感知检索：
  - 内部层级社区（结构社区）
  - 外部主题分类社区（主题社区）

**与我们的方案对比**:

| 维度 | SAT-Graph RAG | 我们的方案 |
|------|--------------|-----------|
| 本体基础 | LRMoo（国际标准） | 自定义三层本体 |
| 层级数量 | 4 层（Norm/Component/Version/Expression） | 3 层（文档/地位/要件） |
| 法律体系 | 巴西法律 | 中国法律 |
| 层级间关系 | 层级包含关系（无谓词） | 37 个谓词的有向边 |
| 合规要素 | 无 | 11 类要件本体（监管主体、合规行为等） |
| 检索方法 | 社区过滤 + 语义检索 | 双路径召回 + 多智能体探索 |

**关键差异**:
- SAT-Graph RAG 聚焦于**文档内部的层级结构**（编章节条），不建模法律文件之间的效力层级关系
- SAT-Graph RAG 没有"地位层"（宪法 > 法律 > 行政法规）的概念
- SAT-Graph RAG 没有"要件层"（监管主体、合规行为等），只关注文档结构
- 两者的"多层"含义不同：SAT-Graph 是文档内部的多层，我们是跨文档的多层

**置信度**: 中等偏高
- 多层本体建模在法律领域已有先例（LRMoo 是国际标准）
- 但将**文档结构、法律地位、合规要素**三者统一建模为三层本体，未见先例
- SAT-Graph RAG 是最接近的工作，但其"多层"指的是文档内部结构，不是跨文档的效力层级

## 三、创新点 3：分层质量控制（会议主持 Agent + 审计 Agent）

### 最相近的现有工作

**Iterative Audit Convergence (arXiv:2605.12280, 2026-05)**  
*标题: Iterative Audit Convergence in LLM-Managed Multi-Agent Systems: A Case Study in Prompt-Engineering Quality Assurance*

**方法概述**:
- LLM 驱动的迭代式审计系统（AEGIS）
- 7 个 Agent 通道（lanes）协作完成软件工程任务
- 迭代审计直到收敛（无新发现）
- 有收敛判据（convergence criterion）
- 有缺陷分类法（7 类缺陷）

**与我们的方案对比**:

| 维度 | Iterative Audit Convergence | 我们的方案 |
|------|---------------------------|-----------|
| 应用领域 | 软件工程（Jira 工单） | 法律合规检索 |
| 审计对象 | 软件规格文档的缺陷 | 检索 Agent 的探索质量 |
| 审计层数 | 单层（审计 Agent 直接审计） | 两层（会议主持 + 审计 Agent） |
| 收敛判据 | 全范围审计无新发现 | 无兜圈子且上下文补全 |
| 多 Agent 协作 | 7 个功能通道 | n 个检索 Agent + 会议主持 + 审计 |

**关键差异**:
- 该论文的"审计"是审计**软件规格文档的质量**，不是审计**检索过程的质量**
- 该论文是单层审计（审计 Agent 直接审计文档），我们是分层审计（会议主持粗筛 + 审计 Agent 细筛）
- 该论文的"收敛"是指审计过程收敛，我们的"兜圈子检测"是指检测检索 Agent 的无效探索

**其他相关工作**:
- SPLIT-RAG、PAI-2、Graph Counselor（已读全文）：多智能体 GraphRAG，但无独立审计层
- Think-on-Graph 3.0 (arXiv:2509.21710): 多智能体 GraphRAG，但无审计机制

**置信度**: 中等
- 在 GraphRAG 领域，分层审计机制未见先例
- 但在软件工程领域，迭代审计和收敛判据已有工作（如本论文）
- 我们的"分层"设计（会议主持 + 审计 Agent）在 GraphRAG 中较新，但"审计"概念本身不新

## 五、建议

### 5.1 专利撰写建议

1. **强调"跨文档"的多层建模**
   - 明确区分我们的"地位层"（宪法 > 法律 > 行政法规）与 SAT-Graph RAG 的"文档内部层级"（编 > 章 > 节 > 条）
   - 强调 37 个谓词的精确定义

2. **突出中国法律体系特化**
   - 12 类法律文件的完整定义
   - 12×12 谓词矩阵的完备性
   - 这是领域知识的形式化，具有技术壁垒

3. **明确"分层审计"的设计动机**
   - 会议主持 Agent 做粗筛（范围适用性）
   - 审计 Agent 做细筛（兜圈子检测）
   - 两层设计的优势：减少审计开销，提高审计精度

### 5.2 补充查新建议

1. **中文数据库检索**
   - CNKI、万方：检索"法律知识图谱"、"中国法律体系本体"、"法律效力层级"
   - 确认 37 谓词矩阵在中文文献中是否已有

2. **法律信息学领域检索**
   - Legal Informatics、Artificial Intelligence and Law 等期刊
   - 确认多层本体建模在法律领域是否有更早的工作

3. **软件工程领域检索**
   - 确认"分层审计"在软件质量保证中是否有先例

## 六、结论

基于 arXiv 检索的 64 篇论文和 5 篇深入分析，**三个创新点均具有一定的新颖性**：

1. **三层本体统一建模**：SAT-Graph RAG 是最接近的工作，但其"多层"是文档内部结构，不是跨文档的效力层级。我们的方案在**跨文档建模**方面有创新。

2. **37 谓词矩阵**：在已检索的论文中未见类似工作，具有较强的新颖性。但需补充中文数据库检索。

3. **分层质量控制**：在 GraphRAG 领域未见分层审计机制，但在软件工程领域有迭代审计工作。我们的"分层"设计（会议主持 + 审计 Agent）在 GraphRAG 中较新。

**总体评价**: 本专利方案在法律领域的 GraphRAG 系统中具有明确的创新点，特别是在**中国法律体系的形式化建模**（37 谓词矩阵）方面具有较强的技术壁垒。建议补充中文数据库检索以进一步验证。
