\begin{abstract}

Graph Chain-of-Thought (Graph-CoT) enables large language models (LLMs) to perform step-by-step reasoning over graph-structured knowledge, but existing pipelines suffer from low accuracy, excessive token usage, high latency, and low throughput due to single-agent monolithic prompts, repeated context re-encoding, and inefficient serving execution. 
%
We present \projectname, the first multi-agent Graph-CoT system co-designed with an optimized LLM serving architecture. \projectname decomposes reasoning into specialized agents for classification, reasoning, action generation, and graph retrieval, enabling branching and selective context sharing to reduce prompt length and reasoning iterations while preserving reasoning quality, thereby improving accuracy and reducing overall token consumption. 
%
To scale inference, we introduce a Graph-CoT–aware LLM inference mechanism with graph-specific KV-cache management, priority-based eviction, and pipelined execution to improve serving efficiency. 
%
Experiments demonstrate that \projectname improves answer accuracy by up to \textbf{38\%}, reduces token cost by up to \textbf{95.7\%}, lowers inference latency by \textbf{90.3\%}, and achieves up to \textbf{15.1$\times$} higher throughput compared to state-of-the-art Graph-CoT baselines, enabling efficient adoption for complex real-world reasoning at scale.


%
% Graph Chain-of-Thought (Graph-CoT) reasoning, which combines graph-based Retrieval Augmented Generation (RAG) with step-by-step inference using Large Language Models (LLMs), has emerged as a promising paradigm for improving LLM performance on structured knowledge tasks. However, existing Graph-CoT frameworks primarily adopt a single-agent architecture, which introduces two major limitations: (1) \textit{low reasoning accuracy and high token overhead}, as the entire reasoning trajectory must be serialized into a single monolithic prompt; and (2) \textit{high end to end latency and low throughput}, due to inefficient key-value (KV) cache utilization and the lack of pipelined execution in current LLM serving systems.

% In this paper, we present \projectname, the \textbf{first multi-agent Graph-CoT reasoning framework co-designed with system-level optimizations for efficient LLM inference and graph retrieval}. \projectname decomposes complex reasoning tasks into four specialized agents: a Graph RAG retriever and three LLM-based agents, including a classification agent, a reasoning agent, and an action agent. This modular design enhances reasoning accuracy while significantly reducing token usage, as each agent operates on a focused, task-specific context. To further improve system efficiency, \projectname integrates KV cache management and pipelined execution through three core optimizations: (1) a \textit{vertex-centric KV cache reuse model} that leverages graph structure to maximize cache sharing across agents and sessions; (2) a \textit{priority-based KV cache eviction policy} that increases cache hit rates under constrained memory budgets; and (3) a \textit{pipelined execution strategy} that overlaps graph retrieval with LLM decoding to reduce latency and boost throughput. Experimental results on a comprehensive graph reasoning benchmark demonstrate that \projectname improves answer accuracy by up to \textbf{38\%}, while also reducing token cost by up to \textbf{95.7\%}, lowering inference latency by \textbf{90.3\%}, and achieving up to \textbf{15.1$\times$} higher throughput compared to state-of-the-art single-agent Graph-CoT baselines.
\end{abstract}
