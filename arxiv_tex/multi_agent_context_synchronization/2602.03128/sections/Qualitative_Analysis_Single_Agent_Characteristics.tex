We begin by contrasting the \emph{main purpose} each framework assigns to an agent and the corresponding \emph{agent architecture}, which together determine the role an agent plays within the system. Graph-based frameworks are designed for deterministic workflow orchestration, where agents function as execution units embedded in explicit control graphs that prescribe when and how computation proceeds. Role-based frameworks instead target task-driven coordination, organizing agents around specialized roles and delegation mechanisms that enable flexible task decomposition and collaborative problem solving. In contrast, Generative Agent-Based Modeling (GABM) frameworks aim to simulate emergent behavior through environment-mediated agent interaction, treating agents as autonomous actors whose behavior unfolds through repeated perception--action cycles within a shared environment. In systems such as Concordia, agent reasoning is further supported by an internal component network inspired by cognitive and social modeling~\cite{minsky1988society}, which maintains long-term state and contextual information to guide behavior over extended interactions~\cite{vezhnevets2023generative}.
