\section{Conclusion}
\label{sec:conclusion}

% This work presents the first controlled, architecture-focused empirical study of multi-agent LLM frameworks. It shows that system behavior is governed by framework execution design rather than by model capability alone. Through an architectural taxonomy and the unified MAFBench benchmark suite, we isolate how orchestration overhead, memory semantics, planning interfaces, specialization mechanisms, and communication topology shape latency, robustness, accuracy, and scalability under fixed models and tasks. The results demonstrate that throughput follows orchestration structure in practice. Memory effectiveness depends on retrieval and accumulation semantics. Planning outcomes are driven by interface constraints. Specialization behavior is determined by procedural conditioning. Coordination dynamics are dictated by communication structure. Together, these findings establish multi-agent frameworks as execution architectures whose interfaces control information flow and system performance. They motivate principled architectural design and standardized evaluation as prerequisites for building scalable and reliable multi-agent AI systems.

% This paper addresses a growing gap between the rapid proliferation of multi-agent LLM frameworks and the lack of principled methods to understand how architectural design choices shape system behavior. Existing benchmarks evaluate isolated agent capabilities but fail to isolate framework-level effects on execution cost, robustness, and scalability. We introduced an architectural taxonomy that systematically characterizes multi-agent frameworks along core dimensions of orchestration, memory, planning, specialization, and coordination, and applied it to widely used systems. Building on this taxonomy, we presented \textbf{MAFBench}, a unified evaluation suite that enables controlled, architecture-level comparison under fixed models and tasks. Our empirical study shows that system performance is governed primarily by execution semantics and interface design rather than model quality or interaction budget, revealing consistent trade-offs across orchestration depth, memory architecture, planning interfaces, and communication topology. These findings expose fundamental limitations in current frameworks and benchmarks, particularly around memory revision, coordination scalability, and automation. We believe this work establishes a foundation for treating multi-agent system design as a systems problem, and motivates future research on adaptive architectures, principled coordination control, and automated compilation of agentic systems from high-level specifications.
