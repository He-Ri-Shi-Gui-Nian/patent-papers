% As summarized in Table~\ref{tab:framework-comparison}, contemporary multi-agent frameworks primarily rely on retrieval-based long-term memory (LTM; persistent semantic recall) and bounded short-term context (STM; recent interaction history), while offering limited or no native support for entity memory (EM; structured facts about entities), update-capable working memory (WM; mutable internal state), or first-class external knowledge (EK; attached documents or files)~\cite{langgraph2025,crewai2025,wu2024autogen,openaiagents2025}. The key distinction across framework families lies in how these storage components are exposed and operationalized during execution rather than in fundamentally different memory paradigms. Graph-based frameworks adopt a lightweight, developer-controlled model in which optional retrieval-based LTM is complemented by bounded STM through message accumulation or explicit state variables propagated along the execution graph, with EM, WM, and EK handled procedurally by the developer~\cite{langgraph2025}. Role-based frameworks extend this design by supporting persistent, retrieval-centric LTM scoped to agents or tasks and optional EM via embedding-based similarity search, alongside RAG-style STM over recent interactions; however, memory access remains governed by relevance or recency rather than explicit update or deletion semantics~\cite{crewai2025,wu2024autogen,openaiagents2025}. In contrast, GABM frameworks maintain memory at the environment level, where stored observations are queried indirectly to update component networks (developer-defined, stateful modules with explicit update functions) that serve as derived working memory, without exposing STM, EM, or EK as agent-addressable abstractions~\cite{vezhnevets2023generative}. Overall, storage across all framework families prioritizes recall and contextual reconstruction over controlled modification, a design choice that underpins the memory behaviors examined in Section~\ref{subsec:memoryagentbench}.

We consider memory abstractions exposed by existing frameworks, including long-term memory (LTM), short-term memory (STM), entity memory (EM), working memory (WM), and external knowledge (EK). Graph-based frameworks primarily expose retrieval-based long-term memory with bounded short-term execution state, while entity memory, working memory, and external knowledge are managed procedurally by developers~\cite{langgraph2025}. Role-based frameworks follow a similar retrieval-centric design but may additionally support structured entity memory via embedding-based retrieval and explicit attachment of external knowledge, such as documents or files~\cite{crewai2025,wu2024autogen,openaiagents2025}.
