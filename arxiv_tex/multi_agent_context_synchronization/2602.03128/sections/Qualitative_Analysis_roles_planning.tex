



% \subsubsection{Role, Goals \& Planning}
% In multi-agent frameworks, \emph{roles} condition agent behavior, \emph{goals} define intended objectives, and \emph{planning} determines how future actions are structured to achieve those objectives~\cite{castelfranchi1998modelling,crewai2025,wu2024autogen,openaiagents2025}. Frameworks differ in whether these concepts are exposed as explicit abstractions or remain implicit within prompts and execution logic, and in how strongly they constrain runtime behavior~\cite{sang2025beyond}. Graph-based frameworks encode roles, goals, and plans procedurally within developer-defined execution graphs, where agent behavior is determined by graph position, prompts, and static control flow rather than first-class role or goal objects~\cite{langgraph2025,langflow2025,autogpt2025,n8n2025}. Role-based frameworks elevate roles and goals to explicit configuration elements that guide reasoning, tool access, and interaction, with planning typically expressed as natural-language task plans or manager--worker coordination rather than rigid action sequences~\cite{crewai2025,wu2024autogen,openaiagents2025,OpenAgents}. In contrast, Generative Agent-Based Modeling frameworks do not rely on task-oriented roles or explicitly specified goals; instead, behavior and planning emerge implicitly from repeated agent--environment interactions within a perception--action loop, with any role-like descriptions functioning as optional personas rather than enforced responsibilities~\cite{vezhnevets2023generative}. Together, these design choices determine how explicitly agent behavior is specified versus allowed to emerge at runtime.
